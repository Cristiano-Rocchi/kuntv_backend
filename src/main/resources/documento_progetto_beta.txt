*BETA
website streaming video di serie animate, niente registrazione, accesso al sito tramite codice segreto. sezione per gli admin con codice segreto diverso dallo user.
i video saranno raccolti in sezioni(esempio i simpson), e in stagioni(se presenti).
le SEZIONI hanno un titolo obbligatorio(i simpson), un tag obbligatorio(Generi), l'anno o gli anni in cui è stata trasmessa(esempio "2010-in corso") e un immagine di copertina obbligatoria.
Le STAGIONI hanno un titolo e l'anno.
i VIDEO hanno un titolo obbligatorio(esempio puntata 1, puntata 2...), una durata in stringa obbligatoria (esempio "35 min") e un file video obbligatorio

Gli UTENTI hanno la possibilità di vedere tutto il catalogo dei video
Gli ADMIN hanno la possibilità di vedere caricare modificare o eliminare video,stagioni o sezioni tramite una pagina dedicata.

RICERCHE
possibilità di cercare singoli video, sezioni o stagioni.
la ricerca principale sara solo delle sezioni(esempio ricerca "simpson"), dentro la sezione sara possibile cercare per stagioni(esempio "stagione 1"), nelle stagioni sara possibile cercare i singoli episodi("puntata 1").
Ricerche abilitate sia per nome tramite form o per nome tramite dropdown

GESTIONE IMMAGINI E VIDEO
la gestione delle immagini sara affidata a cloudinary mentre per la gestione dei video vengono caricati su Google Drive tramite un sistema multi-account (16 account iniziali, con la possibilità di aggiungerne altri in futuro). Il backend gestisce l'autenticazione tramite credenziali OAuth di Google per ogni account, e i file vengono distribuiti tra gli account in base allo spazio disponibile.





***********************************ENTITIES*****************************************************

-SEZIONI- (titolo, fotoUrl, tag, anno, Lista Stagioni[], Lista Video[])

-STAGIONI- (titolo, anno, Lista Video[], Sezione associata)

-VIDEO- (titolo, durata, fileLink, Stagione associata, Sezione associata)



**********************************ENDPOINT*******************************************************
LOGIN con chiave segreta

GET aperto a tutti gli user

POST,PUT,DELETE solo per admin
per la creazione oggetti seguire il flusso delle correlazioni tra tabelle(Sezione,Stagioni,video)

---------------------------POST-------------------------------

LOGIN
/auth/login
{
"secret" :[inserire segreto]
} 


SEZIONI
/api/sezioni                  
{
  "titolo": "Sezione Esempio",
  "fotoUrl": "https://example.foto.com",
  "tag": "azione",
  "anno": "2000-incorso"
}

STAGIONI
 api/stagioni
{
  "titolo": "Stagione Esempio",
  "anno": "2024",
   "sezione": {
    "id": "uuid della sezione"
  }
  
}

VIDEO
 api/video?stagioneId={inserire id stagione}
{
  "titolo": "Video Esempio",
  "durata": "30:00",
  "fileLink": "https://examplevideo.com/video.mp4"
}


------------------------------PUT-------------------------
SEZIONI
api/sezioni/{sezioneID}
body uguale a post

STAGIONI
api/stagioni{stagioneID}
body uguale a post

VIDEO
api/video{videoID}
{
  "titolo": "Video Aggiornato",
  "durata": "1h 30min",
  "fileLink": "https://esempio.com/nuovo_video.mp4",

  }

----------------------------DELETE----------------------------
Attenzione eliminando una sezione o stagione si elimina tutto cio che c'è associato ad essa

SEZIONI
api/sezioni/{sezioneID}

STAGIONI
api/stagioni{stagioneID}

VIDEO
api/video{videoID}






********************************GESTIONE DELLE ECCEZIONI********************************************

controllo delle eccezioni con messaggi personalizzati. gestite con un exceptionshandler e classi eccezioni.

ERRORI GESTITI:
-errore nel digitare l'endpoint
-errore nel inserimento del secret per accedere
-errore nella creazione degli oggetti(sezioni,stagioni,video)
-errore nel recupero degli oggetti specifici qual'ora l'id inserito nell endpoint fosse sbagliato( esempio /api/video/7huhd7838 <---id sbagliato)
